Описание игры
ConsoleRPG — это классическая консольная ролевая игра, написанная на C++ с использованием объектно-ориентированного программирования. Игрок управляет героем, исследует подземелья, сражается с монстрами и собирает снаряжение.

Ключевые особенности:

⚔️ Боевая система с пошаговыми сражениями

🧙 Классы персонажей (воин, маг, лучник)

🏹 Система предметов (оружие, броня, зелья)

🧭 Генерация подземелий

📊 Система прокачки и развития персонажа

💾 Сохранение прогресса

🚀 Запуск игры
Требования для запуска:
Компилятор C++17 (GCC, Clang, MSVC)

CMake 3.10+

Git (для клонирования репозитория)

Инструкция по запуску:
bash
# 1. Клонирование репозитория
git clone https://github.com/ChocoChocobo/ConsoleRPG.git
cd ConsoleRPG

# 2. Сборка проекта
mkdir build && cd build
cmake ..
make

# 3. Запуск игры
./ConsoleRPG
Альтернативный способ (Windows):

Откройте проект в Visual Studio

Соберите решение (Ctrl+Shift+B)

Запустите исполняемый файл

🏗️ Основные модули игры
1. Ядро игры (Core Module)
Модуль отвечает за:

🎯 Управление игровым циклом

💾 Сохранение/загрузку игры

🎮 Обработку ввода пользователя

📊 Логирование событий

cpp
class Game {
    void Run();           // Главный цикл
    void SaveGame();      // Сохранение
    void LoadGame();      // Загрузка
    void ProcessInput();  // Обработка ввода
};
2. Система сущностей (Entities System)
Реализует паттерн Наследование для создания игровых объектов:

text
ICharacter (интерфейс)
    ↑
Character (базовый класс)
    ├── Player (игрок)
    ├── Enemy (враг)
    │   ├── Goblin (гоблин)
    │   ├── Dragon (дракон)
    │   └── Boss (босс)
    └── NPC (торговцы, квестодатели)
3. Инвентарь и предметы (Inventory System)
Система предметов использует:

🎒 Контейнеры STL (std::map, std::vector)

🏷️ Интерфейсы для универсального взаимодействия

🔧 Фабричный метод для создания предметов

Пример использования std::map для инвентаря:

cpp
std::map<ItemType, std::vector<std::unique_ptr<IItem>>> inventory;

// Добавление предмета
inventory[ItemType::WEAPON].push_back(
    std::make_unique<Sword>("Экскалибур", 50)
);
4. Боевая система (Combat System)
Особенности боевой механики:

⏳ Пошаговые сражения

⚖️ Расчет урона с учетом характеристик

🛡️ Система защиты и сопротивлений

🎲 Рандомные критические попадания

🎯 Цели разработки
Образовательная: Изучение ООП и паттернов проектирования

Практическая: Применение STL контейнеров (std::map, std::vector)

Архитектурная: Создание модульной и расширяемой системы
